# Vault 폴더 구성 가이드

> 서랍 없는 책상을 상상해보자.
> 영수증, 계약서, 메모지, 명함이 한 곳에 쌓여있으면 — 필요할 때 찾을 수가 없다.
> vault의 폴더는 **"이런 정보는 여기"** 라는 약속이다.
> 한 번 익혀두면, AI가 알아서 맞는 서랍에 넣어준다.

---

## 폴더 구조 전체 지도

```
vault/
├── 00_inbox/          ← 일단 다 여기. 판단은 나중에
├── 10_daily/          ← 오늘 하루의 기록
├── 20_depts/          ← 부서별 실무 지식
├── 30_cross_functional/  ← 팀 공동 프로젝트
├── 40_resources/         ← 배운 것, 쓸 것, 참고할 것
├── 50_personal/          ← 개인 업무
├── 70_decisions/         ← 중요한 결정의 기록
└── 90_meta/              ← 시스템이 알아서 관리
```

---

## 00_inbox/ — "일단 다 여기에"

### 이게 뭔가요?
**생각하지 말고 넣는 곳.** 좋은 기사를 봤다, 중요한 메일이 왔다, 메모를 남기고 싶다 — 일단 inbox에 던져두면 된다. 정리는 AI가 도와준다.

### 나는 이렇게 씁니다
```
좋은 뉴스 기사 발견 → 브라우저 확장으로 저장 → inbox/web/에 자동 보관
중요한 메일 → capture_outlook → inbox/claude/에 자동 저장
```

### 이게 왜 좋은가요?
- "나중에 정리하자"는 늘 미뤄진다. inbox에 넣는 것만 습관으로 만들면, **정리는 AI가 알아서** 해준다.
- 지금 바쁠 때 → 일단 inbox. 여유 생기면 → "inbox 정리해줘" 한 마디.

```
[써볼 수 있는 프롬프트]
"내 inbox에 뭐가 쌓여있어?"
"이번 주 inbox에서 내 업무랑 관련된 것만 골라줘."
"이 링크 내용 요약해서 inbox에 넣어줘."
```

---

## 10_daily/ — "오늘 하루의 기록"

### 이게 뭔가요?
**하루 단위 메모장.** 오늘 있었던 일, 결정한 것, 생각난 것을 날짜별로 기록한다. 나중에 "그날 무슨 얘기 했더라?" 할 때 찾아보는 폴더.

### 나는 이렇게 씁니다
```
매일 아침 자동으로 브리핑 파일 생성 →
  오늘 주요 뉴스 + 현안 + 할 일이 이미 정리된 상태로 출근
```

### 이게 왜 좋은가요?
- 업무 일지를 따로 쓰지 않아도, 매일의 기록이 날짜별로 자동 쌓인다.
- "지난주 월요일에 뭐 했더라?" → AI한테 물어보면 바로 답이 나온다.

```
[써볼 수 있는 프롬프트]
"오늘 있었던 미팅 내용 일일 노트로 정리해줘."
"이번 주 내가 처리한 업무 목록 만들어줘."
"지난 화요일 회의에서 결정된 게 뭐였지?"
```

---

## 20_depts/ — "부서별 실무 지식"

### 이게 뭔가요?
**정제된 실무 지식의 정본.** inbox에서 걸러진 진짜 가치 있는 정보가 여기 자리잡는다. 내 담당 부서 폴더에, 내가 반복적으로 참조하는 정보들을 모아두는 곳.

```
20_depts/
├── finance/     ← 재무·회계 관련
├── hr/          ← 인사·노무
├── it/          ← IT·시스템
├── marketing/   ← 마케팅·프로모션
├── purchasing/  ← 구매·발주
└── strategy/    ← 전략·경영
```

### 나는 이렇게 씁니다
```
세금계산서 분석 결과 → finance/ 노트로 저장
자금계획 업데이트 → finance/자금계획/ 에 반영
더존 이슈 원인 분석 → finance/2026-06-24_더존-이슈.md
```

### 이게 왜 좋은가요?
- "이거 전에도 한 번 봤는데..." 를 없앤다. 한 번 정리한 건 여기 있으니까.
- **부서가 다른 사람도 내 폴더 볼 수 있다** — "finance팀은 이걸 어떻게 처리하나?" 참조 가능.

```
[써볼 수 있는 프롬프트]
"내가 자주 참조하는 [구매팀 발주 절차]를 노트로 정리해줘.
 폴더: 20_depts/purchasing/"

"지난 달에 처리한 [HR 취업규칙 건] 내용 다시 찾아줘."

"[IT팀] 관련 노트 중 아직 해결 안 된 이슈가 뭐가 있어?"
```

## 30_cross_functional/ — "팀이 함께 쓰는 공간"

### 이게 뭔가요?
**여러 부서가 함께 만드는 자산.** 특정 부서 혼자의 것이 아닌, 팀 전체가 참여하는 프로젝트가 여기 산다. 지금 이 문서가 있는 폴더가 바로 여기다.

```
30_cross_functional/
└── PHK-AI교육/   ← 지금 우리가 함께 만들고 있는 것
```

### 이게 왜 좋은가요?
- Finance팀 AI 교육 자료처럼, **누군가 혼자 만들기엔 아깝고 팀 전체에 쓸모 있는 것**들이 여기서 쌓인다.
- 한 명이 만든 프롬프트를 팀 전체가 쓸 수 있다.

```
[써볼 수 있는 프롬프트]
"PHK-AI교육 자료 중 내가 써보고 싶은 프롬프트 추천해줘."

"팀에서 공유할 만한 엑셀 자동화 팁을
 PHK-AI교육 폴더에 정리해줘."

"내가 직접 써본 AI 활용 사례를
 팀 공유 노트에 추가해줘."
```

---

## 40_resources/ — "배운 것, 쓸 것, 참고할 것"

### 이게 뭔가요?
**재사용 가능한 지식 창고.** 책을 읽고 남긴 노트, 유용한 프롬프트 모음, 업무에 쓸 수 있는 템플릿. "언젠가 다시 쓸 것 같은 것들"이 여기 있다.

### 이게 왜 좋은가요?
- 좋은 글을 읽었을 때 밑줄만 긋고 끝나는 게 아니라, **실제로 내 업무에 연결**된다.
- 한 번 정리한 프롬프트를 다음에 또 쓸 수 있다.

```
[써볼 수 있는 프롬프트]
"이 기사 읽었는데, 내 업무에 쓸 수 있는 부분만 3줄로 요약해줘.
 resources/learning에 저장해줘."

"내가 자주 쓰는 엑셀 수식 설명 프롬프트를
 resources/prompts/에 저장해줘."

"지금까지 내가 정리한 학습 노트 목록 보여줘."
```

## 50_personal/ — "개인 업무"

### 이게 뭔가요?
업무와 분리된 **개인 영역.** 시스템에서 자동 스캔 제외.

---

## 70_decisions/ — "왜 그 결정을 했는가"

### 이게 뭔가요?
**중요한 결정의 기록.** "그때 왜 그렇게 결정했지?"를 나중에 설명할 수 있게 해준다. 계약, 방향 전환, 팀 결정 같은 것들이 여기 남는다.

### 이게 왜 좋은가요?
- 결정을 기록해두면 **같은 고민을 두 번 안 해도 된다.**
- 새로운 팀원이 왔을 때 "우리가 왜 이렇게 하고 있냐"를 문서로 설명할 수 있다.

```
[써볼 수 있는 프롬프트]
"오늘 결정한 [발주 방식 변경] 건을
 결정 노트로 기록해줘.
 내용: 배경·선택지·결정 이유."

"지난 분기 우리 팀이 내린 주요 결정들 목록 만들어줘."
```

---

## 90_meta/ — "시스템이 알아서"

### 이게 뭔가요?
vault가 스스로 관리하는 영역. 직접 건드리지 않아도 된다.

---

## 폴더를 나눠두면 뭐가 좋은가요?

폴더를 나눠두면 단순히 "정리된 느낌"이 아니라 **AI가 더 똑똑하게 동작한다.**

| 상황 | 폴더 없을 때 | 폴더 있을 때 |
|---|---|---|
| "지난달 세금계산서 건 찾아줘" | 전체 파일 뒤짐 | `finance/`만 검색, 즉시 찾음 |
| "팀에 공유할 자료 있어?" | 뭐가 공유용인지 모름 | `cross_functional/` 바로 확인 |
| "내가 내린 결정 기록 있어?" | 일일노트·메모 다 뒤져야 함 | `70_decisions/` 바로 확인 |
| 새 팀원에게 인수인계 | 구두로 설명해야 함 | 폴더 보여주면 됨 |

> **폴더 = AI에게 주는 맥락.**
> "이건 finance 관련이야" 라고 폴더로 알려주면, AI가 답변할 때도 그 맥락으로 답한다.

---

## 처음 시작한다면?

한꺼번에 다 쓸 필요 없다. 이 순서로 시작하면 된다.

```
1단계: 00_inbox에 일단 저장하는 습관
   → "이 기사 inbox에 넣어줘"

2단계: 내 부서 폴더에 반복 참조하는 정보 하나 만들기
   → "내가 자주 쓰는 [구매 절차]를 노트로 정리해줘"

3단계: 팀 공유 폴더에 기여하기
   → "내가 써본 AI 활용 사례 PHK-AI교육 폴더에 추가해줘"
```
