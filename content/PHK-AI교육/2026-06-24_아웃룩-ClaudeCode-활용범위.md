# 아웃룩 + Claude Code — 어디까지 됩니까
_PHK-AI 교육 | 기술 통합 심화편 | 2026-06-24_

> **대상**: 이 연결을 직접 구현하고 싶은 Finance팀 누구나  
> **참고 파일**: `90_meta/ethan-email-style-guide.md` · `scripts/capture_outlook.py` · `scripts/open_outlook_draft.py`

---

## 다섯 줄 요약

1. 아웃룩 메일이 매일 자동으로 텍스트 파일(.md)로 저장된다 (현재 24,000통 누적)
2. Claude Code에게 말하면 → 아웃룩 초안 창이 자동으로 열린다
3. 중간에 막히는 단계(패키지 오류)는 Claude Code가 스스로 해결한다
4. 저장된 메일에서 프로젝트별 맥락을 자연어로 즉시 꺼낼 수 있다
5. 발송 이력 334통 분석 → 내 말투 그대로 초안이 나오게 학습시켰다

---

## ① 메일 → .md 자동 저장

### 원리

Python 스크립트(`scripts/capture_outlook.py`)가 매일 자동 실행됩니다.  
Outlook COM 인터페이스를 통해 받은 메일을 읽고, 각각 텍스트 파일로 변환합니다.

```
# Windows 작업 스케줄러가 매일 18:00 자동 실행
python scripts/capture_outlook.py

# 저장 경로 예시
vault/archive/2026/06/2026-06-04_GES-정산-확인요청.md
vault/archive/2026/06/2026-06-11_손익업데이트-CFO보고.md
vault/archive/2026/06/2026-06-18_iLumen-셋업-현황공유.md
```

### 저장되는 파일 형태

```markdown
---
date: 2026-06-04
from: Jason부장님 <jason@pizzahutkorea.com>
to: Ethan.Yoon@pizzahutkorea.com
subject: GES 정산 확인 요청 건
---

안녕하세요, Ethan님

이번 달 GES 정산 내역 첨부드립니다. 확인 부탁드립니다.

감사합니다.
Jason.
```

### 왜 .md 저장인가

아웃룩 검색 = 키워드가 **정확히** 기억나야 함.  
vault + Claude Code = **자연어**로 물어보면 됨.

| 상황 | 아웃룩 검색 | vault + Claude Code |
|---|---|---|
| "3개월 전 거래처 A 단가 건이 어떻게 됐더라?" | 키워드 기억 안 나면 못 찾음 | "거래처 A 단가 관련 메일 찾아줘" → 즉시 |
| "이 프로젝트 처음부터 타임라인 정리" | 수동으로 하나씩 읽어야 함 | "iLumen 관련 메일 날짜순 정리" → 즉시 |
| "인수인계 자료 — 내가 어떤 약속 해놨지?" | 중요 스레드 하나씩 찾아야 함 | "내가 보낸 메일 중 약속·커밋 포함된 것" → 즉시 |

## ② 명령어 한 줄 → 아웃룩 초안 창이 열린다

### 작동 방식

Claude Code에게 말로 요청하면:

1. 메일 초안 내용을 작성하고
2. `scripts/_draft_tmp.json` 파일을 생성하고
3. `python scripts/open_outlook_draft.py`를 실행합니다
4. **아웃룩 초안 창이 자동으로 열립니다**

### 실제 프롬프트 예시

```
【기본 — 단순 요청】
Jason부장님한테 이번 달 GES 정산 확인 요청 메일 써서
아웃룩 초안으로 띄워줘. 정중하고 짧게.
```

```
【상세 — 수신자·제목·내용 지정】
수신: Jason부장님
참조: Daniel실장님
제목: 이번 달 정산 내역 공유 건
내용: 첨부 파일 공유드린다는 한 줄 + 확인 부탁
아웃룩 초안으로 열어줘.
```

```
【영문 — 글로벌 팀 발송】
글로벌 팀(Hi all)에게 iLumen 셋업 진행 상황 업데이트 메일을
영문으로 써서 아웃룩 초안 열어줘. 2문단 이내로 간결하게.
```

```
【답장 초안 — 받은 메일 붙여넣기】
아래 메일에 "7월 협의 일정 잡겠습니다"로 답장 초안 써줘.
[메일 내용 붙여넣기]
```

```
【협조 요청 공지】
Finance팀 전체에게 새 경비 정산 프로세스 안내 메일을 써줘.
배경 2줄 + 변경사항 불릿 + 시행일 + 협조 부탁으로.
아웃룩 초안으로 열어줘.
```

### 핵심 제약 — 발송은 항상 사람이

`open_outlook_draft.py`는 `mail.Display(False)`만 호출합니다.  
`Send()`는 코드에 없습니다. 초안 창이 열리면 직접 확인 후 발송 버튼을 누릅니다.

## ③ 중간에 막혀도 Claude Code가 스스로 해결

### 패키지 자동 설치 흐름

아웃룩 자동화에 `pywin32` 패키지가 필요합니다.  
처음에 없으면 이런 오류가 납니다:

```
ImportError: No module named 'win32com'
```

Claude Code가 자동으로 진단합니다:

> "아웃룩 자동화에 `pywin32` 패키지가 필요합니다. 설치할까요?"  
> → 승인 한 번 → `pip install pywin32` 자동 실행 → 계속 진행

### 이게 중요한 이유

- **에러 원인을 몰라도 됩니다.** Claude Code가 진단·처방합니다.
- **승인은 한 번.** 이후엔 묻지 않고 바로 실행합니다.
- **어떤 패키지도 동일.** pandas·openpyxl·requests·pdfplumber 전부 이 방식.
- **기술적 단절이 없습니다.** "계속 진행해줘"만 하면 됩니다.

### 실제 경험

처음 `capture_outlook.py` 만들 때 pywin32 → pydantic → python-dateutil 순서로  
세 번 패키지 설치가 필요했습니다. 세 번 모두 Claude Code가 감지·설치·재시도를  
자동으로 했고, 제가 한 것은 "승인" 버튼 세 번이 전부였습니다.

## ④ 프로젝트별 메일 맥락 — 3개월 전 것도 1분 안에

### 실전 프롬프트 모음

```
【협상 이력 복원】
"GES 정산 관련해서 지난 3개월 간 오고간 메일을 날짜순으로 정리해줘.
 합의된 사항과 아직 결론 안 난 이슈를 구분해서."
```

```
【인수인계 준비】
"iLumen 셋업 관련 IT팀이랑 오고간 메일을 타임라인으로 정리해줘.
 누가 어떤 결정을 했는지 중심으로. 날짜와 의사결정자 포함."
```

```
【클레임·분쟁 대응】
"거래처 A랑 단가 조정 건으로 우리가 어떤 입장을 취해왔는지
 메일 기반으로 정리해줘. 우리 측 주장과 상대 측 주장을 분리해서."
```

```
【상반기 리뷰 준비】
"올해 1월~6월에 CFO에게 보고한 메일들 제목·날짜·핵심 내용을
 표 형태로 만들어줘."
```

```
【팀원 온보딩 지원】
"신입 팀원이 파악해야 할 Finance팀 주요 프로젝트 현황을
 최근 3개월 메일 기반으로 정리해줘."
```

### 실제 사용 경험

입사 초기에 전임자가 어떤 약속을 해놨는지 파악할 때,  
아웃룩 스크롤로 2시간 걸렸던 것을 이 방식으로 5분 만에 정리했습니다.

"양수도 이후 계약 조건 변경 관련해서 오고간 메일 다 찾아줘"라고 물었더니  
날짜순으로 22건이 정리되었고, 그 중 3건에서 중요한 합의 사항을 발견했습니다.

## ⑤ 내 말투를 학습 → 초안이 "내 것처럼" 나온다

### ethan-email-style-guide.md의 탄생

`90_meta/ethan-email-style-guide.md`는 발송 메일 **334통 전수 분석** 결과입니다.  
Claude Code가 패턴을 추출하고, 제가 검토·확정했습니다.

```
# 파일에 담긴 패턴 예시

인사말:   Jason부장님 (이름+직함 붙여쓰기, 띄어쓰기 없음)
팀 공지:  안녕하세요, Team~
서명:     감사합니다. Ethan. (마침표 있음)
영문 서명: Thanks, Ethan
영문 인사: Hi all, (Dear 사용 안 함)

금지어:
  ❌ "항상 수고 많으십니다"
  ❌ "다름이 아니오라"
  ❌ "~하여 주시기 바랍니다" (→ "~부탁드립니다"로)
```

### Before/After 비교

| 상황 | 스타일 가이드 없이 | 스타일 가이드 적용 |
|---|---|---|
| 인사말 | "안녕하세요, Jason 부장님<br>항상 수고 많으십니다." | "안녕하세요, Jason부장님" |
| 서명 | "Best regards,<br>Ethan Yoon" | "감사합니다.<br>Ethan." |
| 부탁 표현 | "확인하여 주시기 바랍니다" | "확인 부탁드립니다." |
| 팀 공지 | "안녕하십니까, 팀원 여러분" | "안녕하세요, Team~" |

### 각자 적용하는 방법

발송 메일 100통 이상이면 동일하게 학습시킬 수 있습니다.

```
Claude Code에게:
"내 아웃룩 sent 폴더에서 최근 6개월 발송 메일을 분석해서
 내 이메일 스타일 가이드 파일을 만들어줘.
 인사말·서명·말투·금지 표현 패턴 중심으로."
```

## 정리

| 기능 | 기술 | 지금 바로 가능? |
|---|---|---|
| 메일 → .md 자동 저장 | Python + Outlook COM | ★★☆ (처음 설정만) |
| 초안 자동 생성 + 창 열기 | 말로 요청 → 초안 창 | ★☆☆ 지금 바로 |
| 패키지 자동 설치 | Claude Code 자동 진단 | ★☆☆ 자동 |
| 프로젝트 맥락 즉시 복원 | 자연어 질문 | ★☆☆ 저장 후 바로 |
| 개인 스타일 학습 | 발송 이력 분석 | ★★☆ (처음 설정만) |

**기술적으로 막히는 단계가 없습니다.**  
Claude Code가 에러를 보고 → 원인 진단 → 패키지 설치 → 재시도까지 스스로 합니다.
