# 아침 파이프라인 — 캡처부터 다이제스트까지

> 정보는 모으는 것보다 **흘러가게 만드는 것**이 핵심.
> 이 파이프라인은 "어제 들어온 정보"가 "오늘 쓸 수 있는 지식"이 되는 경로다.

---

## 전체 흐름 한눈에

```
[AM 5:00 자동]
외부 소스들
  Outlook 메일 · Claude Code 대화 · 웹클리핑
        │
        ▼  capture_all.py (자동 실행)
  00_inbox/ ← 모든 것이 여기 쌓임
        │
        ▼  daily-brief 스킬 (자동 실행)
  10_daily/브리핑A.md ← 오늘의 브리핑 생성
        │
        ▼  git auto-commit (자동 실행)
  변경사항 전체 커밋

[출근 후 — 수동]
  00_inbox/ 항목들
        │
        ▼  triage-inbox 스킬
   버림 / 보관 / 정제 후보 분류
        │
        ▼  distill-note / knowledge-ingest 스킬
  20_depts/ · 30_cross_functional/ · 40_resources/
  (정제 완료 — 진짜 자산이 됨)
        │
        ▼  digest-picks 선별
  외부지식-다이제스트.md ← 오늘의 지식 요약본
```

---

## 1단계: Capture (AM 5:00 자동)

### 무슨 일이 일어나나
컴퓨터가 절전에서 깨어나면 `capture_all.py`가 자동으로 실행된다.

**수집 대상:**
| 소스 | 저장 위치 | 내용 |
|---|---|---|
| Outlook 메일 | `archive/outlook/` | 수발신 메일 전체 |
| Claude Code 대화 | `00_inbox/claude/` | AI와의 작업 세션 |
| 웹 클리핑 | `00_inbox/web/` | 브라우저 확장으로 저장한 기사 |
| 드롭 폴더 | `00_inbox/_drop/` | 수동으로 넣어둔 파일 |

**결과:** 하룻밤 동안 들어온 모든 것이 `00_inbox/`에 정리된 상태로 출근을 맞는다.

### 실제 실행 명령어 (수동으로 돌리고 싶을 때)
```bash
py -X utf8 scripts/capture_all.py
```

### 자동화 설정 (현재 구성)
```
Windows 작업 스케줄러
  작업명: VaultAM5Capture
  실행 시각: 매일 AM 5:00
  절전 해제: ✅ (자동으로 PC를 깨움)
  실행 순서:
    1. capture_all.py    ← inbox 수집
    2. daily-brief 스킬  ← 브리핑A 생성
    3. git commit        ← 전체 커밋
```

---

## 2단계: 브리핑A (AM 5:00 자동 생성)

### 무슨 일이 일어나나
`daily-brief` 스킬이 실행되면 오늘 날짜의 브리핑 파일이 자동 생성된다.

**브리핑A 구성:**
```
10_daily/2026-06-24_브리핑A.md

## 외부 동향 (3줄)
- 쿠팡이츠 AML 10건 급증 (IT 🔴 확인 필요)
- PHK 매각 3Q 최종확정
- 배달앱 공정위 상생안 기각

## 현안 키워드
- iLumen GFM 마감 🔴
- HR 취업규칙 D-4

## 오늘 할 것
- 쿠팡이츠 AML Ryan 이사 확인
- GFM 보고 처리
```

출근해서 열면 **이미 오늘의 맥락이 정리되어 있다.**

### 수동으로 브리핑 만들기
```
Claude Code에서:
"오늘 브리핑A 만들어줘"
또는
/daily-brief
```

---

## 3단계: Triage (출근 후, 수동)

### 무슨 일이 일어나나
`00_inbox/`에 쌓인 것들을 **판단**한다.
이 단계에서 정보의 운명이 결정된다.

**판단 결과 4가지:**
| 판단 | 의미 | 이동 위치 |
|---|---|---|
| 버림 | 가치 없음 | `_processed/trash/` |
| 보관 | 참고용이지만 정제 불필요 | `_processed/` |
| 정제 후보 | 정제하면 자산이 됨 | `distill_candidate: true` 태그 |
| 즉시 액션 | 오늘 처리해야 함 | Outlook 작업 생성 |

### 실제 프롬프트
```
"inbox 트리아지 해줘.
 오늘 들어온 claude/ 와 web/ 폴더 항목들
 버릴 것·보관·정제후보·즉시액션으로 분류해줘."

"이번 주 inbox 중 Finance팀에 관련된 것만
 먼저 트리아지해줘."
```

---

## 4단계: Distill (수동, 가치 있는 것만)

### 무슨 일이 일어나나
triage에서 "정제 후보"로 표시된 것을 **진짜 자산**으로 만든다.
raw 캡처 → 구조화된 노트 → vault 영구 보관.

**Distill 유형:**

| 스킬 | 대상 | 저장 위치 |
|---|---|---|
| `distill-note` | Claude 대화·메모 | `20_depts/` 또는 `40_resources/` |
| `knowledge-ingest` | 외부 논문·기사 URL | `40_resources/learning/` |
| `distill-book` | 독서 캡처 | `40_resources/learning/books/` |

### 실제 프롬프트
```
"이 Claude 대화 내용을
 finance 노트로 distill해줘."

"/knowledge-ingest https://[기사URL]
 → 3버킷(사실/적용/확장)으로 정제"

"이번 주 정제 후보 중
 지금 당장 distill할 것 3개 뽑아줘."
```

---

## 5단계: Digest (수동, 주기적)

### 무슨 일이 일어나나
외부에서 들어온 지식들 중 **오늘 가장 중요한 것**을 선별해서 다이제스트로 만든다.

```
90_meta/_generated/외부지식-다이제스트.md
  ← 오늘의 뉴스·논문 중 pick된 것들 요약
```

### 실제 프롬프트
```
"오늘 web inbox에서 Finance팀에 유의미한
 뉴스 3건 골라서 다이제스트 만들어줘."

"이번 주 knowledge-ledger에 새로 들어온 것 중
 가장 중요한 것 5개 요약해줘."
```

| 시간 | 주체 | 작업 | 도구 |
|---|---|---|---|

| AM 5:10 | 자동 | git 커밋 | `git commit` |
| 출근 직후 | Ethan | 브리핑A 읽기 | `10_daily/` 열기 |

## 팀원이 이 파이프라인에 참여하는 방법

지금 당장 전체 파이프라인을 구축하지 않아도 된다.
**한 단계씩** 편입하면 된다.

### 레벨 1 — 캡처만
```
브라우저 확장(Obsidian Web Clipper)으로
 관심 기사 저장 → 00_inbox/web/ 에 쌓임
```

### 레벨 2 — 트리아지 추가
```
"내가 저장한 web 클리핑들
 이번 주 업무와 관련된 것만 골라줘."
```

### 레벨 3 — Distill 추가
```
"이 기사를 Finance팀 관점에서
 핵심 3줄로 정제해줘. resources에 저장."
```

### 레벨 4 — 팀 다이제스트 기여
```
"내가 정제한 노트 중
 팀에 공유할 만한 것 추천해줘."
```

---

> **핵심 원칙**: 완벽하게 하려다 안 하는 것보다,
> **capture 하나라도 매일 하는 것**이 낫다.
> 파이프라인은 쌓일수록 강해진다.
