# 지적 대화를 위한 AI 언어 수업
_강수진 | 어웨이크(AWAKE) | 완독: 2026-06-24_

---

## 왜 읽었나

AI와 업무·개인 작업을 병행하는 단계에서, 어떻게 프롬프트를 설계해야 더 좋은 결과물을 얻는지에 대한 의문.

---

## 핵심 인사이트

**1. 프롬프트 = AI를 아는 것이 아니라 나를 아는 것**
좋은 프롬프트의 본질은 AI 기술 이해가 아니라, 내가 원하는 것을 명확히 알고 표현하는 능력. AI는 방향을 증폭시킬 뿐 — 방향 설정은 여전히 인간의 몫.

**2. AI는 확률 기계 — 프롬프트 구체성이 출력 품질을 결정**
토큰별 확률 선택 구조상, 역할(페르소나) + 맥락 + 예시 + 출력 형식 지정 네 가지가 원하는 토큰이 선택될 확률을 높인다. 핵심 지시는 프롬프트 끝에 배치 (어텐션이 마지막에 집중).

**3. 할루시네이션 4전략**

| 전략 | 원리 |
|---|---|
| 출처 명시 요청 | "근거 출처 함께 제시해줘" → 확인 가능 데이터로 어텐션 집중 |
| 불확실성 허용 | "모르면 모른다고 말해줘" → '모름' 토큰 선택 확률 상승 |
| 맥락 한정 | "아래 자료만을 바탕으로" → 외부 잘못된 패턴 차단 |
| 생성·검증 분리 | "초안 쓰고 스스로 재검토해줘" → 작성과 비판 프로세스 분리 |

**4. AI 언어 처리 6단계 × 프롬프트 원칙**

| 처리 단계 | 프롬프트 원칙 |
|---|---|
| 토큰화 (한국어 = 영어의 2~3배 소비) | 간결하게, 장황한 어미·블록 반복 줄이기 |
| 임베딩 (의미 좌표) | 정확한 단어 선택 — 유사어도 벡터 좌표가 다름 |
| 어텐션 (문장 내 상호 참조) | 핵심 지시를 **끝에** 배치 |
| 자기회귀 생성 | 구체적일수록 원하는 토큰 선택 확률 ↑ |
| 내부 표상 | 역할 명시 → 기학습 영역 정확히 조명 |
| 할루시네이션 | 출처 요청 + 불확실성 허용 + 생성·검증 분리 |

**5. AI는 구조적 표상을 형성한다 (오셀로 실험, Harvard Li 2023)**
GPT에게 오셀로 기보만 학습 → 내부에 게임판 표상이 자발적으로 형성됨. AI를 단순 패턴 암기기로 과소평가하지 말 것.

## 개념 확장

### Word2Vec — 벡터 공간의 의미 산술

Mikolov et al. (Google, 2013) — Ilya Sutskever, Kai Chen, Greg Corrado, Jeffrey Dean 공동 저자. 단어를 고차원 벡터로 인코딩하여 의미 관계가 벡터 산술로 표현됨을 보였다.

- `King − Man + Woman ≈ Queen` — 의미 관계가 벡터 공간에서 방향(offset)으로 보존
- `Paris − France + Japan ≈ Tokyo` — 국가-수도 관계도 동일한 원리
- 핵심 설계 선택: 비선형 히든 레이어 제거 → 단순하지만 스케일에 강건
- 한계: BERT·Transformer 이전 모델로, 문맥에 무관한 단일 벡터(동음이의어 구분 불가)
- 후계: Transformer의 컨텍스트 임베딩(어텐션)이 이 한계를 극복

> 책의 "임베딩 = 의미 좌표" 설명은 Word2Vec 직관에서 출발했지만, 현재 Claude/GPT는 맥락에 따라 벡터가 달라지는 동적 임베딩을 사용.

---

### 오셀로 GPT 실험 — AI 내부 표상의 실증

**논문**: Li et al. (2023). *Emergent World Representations: Exploring a Sequence Model Trained on a Synthetic Task.* ICLR 2023. [arxiv:2210.13382](https://arxiv.org/abs/2210.13382)

- 게임 기보(수열)만 학습 → 보드 상태를 명시적으로 알려주지 않았음에도, 내부에 비선형 게임판 표상이 발생
- 프로브(probe) 분석으로 중간 레이어에서 보드 상태를 역추적 가능
- 개입 실험(Interventional): 내부 표상을 조작 → 출력 행동이 예측한 방향으로 변화

**후속 논쟁** (Neel Nanda, 기계적 해석가능성 연구자):
- 원논문은 "비선형 표상"이라 했으나, Nanda는 선형 프로브로도 충분히 복원 가능함을 보임
- 함의: 표상의 선형성/비선형성 여부는 열린 질문, 하지만 "표상이 존재한다"는 핵심 결론은 유효

> 실무 적용: AI가 "단순 패턴 암기"가 아니라면 → 역할(페르소나) 부여가 단순 어조 변화가 아니라 내부 표상 자체를 조명하는 효과를 가질 수 있다.

---

### 할루시네이션 방지 — 학술 분류와 책 전략의 대응

불확실성 원천 4분류 (Uncertainty Profiles for LLMs, arxiv:2505.07309):

| 불확실성 유형 | 원인 | 대응 |
|---|---|---|
| Surface Form (SU) | 입력 언어 구조 파싱 실패 | 명확한 문장 구조, 전문 용어 풀어 쓰기 |
| Aleatoric (AU) | 프롬프트 자체의 모호성·불충분 지정 | 역할+맥락+형식 명시 (책의 핵심 주장과 동일) |
| Epistemic (EU) | 모델의 지식·추론 한계 | "모르면 모른다고" 허용, 출처 요청 |
| Operational (OU) | 생성 과정의 논리적 슬립 | 생성·검증 분리 (단계적 재검토) |

책의 4전략(출처 요청 / 불확실성 허용 / 맥락 한정 / 생성·검증 분리)은 EU·OU 유형에 주로 대응. Stanford 2024: RAG + RLHF + 가드레일 조합 시 할루시네이션 96% 감소.
